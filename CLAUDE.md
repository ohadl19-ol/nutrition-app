# CLAUDE.md — אפליקציית תזונה וחיטוב

## מהות הפרויקט
אפליקציית web (PWA) אישית למעקב תזונה, מדידות גוף ואימונים.
שם: "תזונה וחיטוב" / "Nutrition & Shred".

## עקרונות מחייבים
- **קובץ אחד בלבד**: `index.html` (שם הקובץ מאפשר הגשה אוטומטית ב-GitHub Pages) — כל ה-HTML, CSS ו-JavaScript בקובץ עצמאי יחיד. ללא שרת, ללא build, ללא תלות באינטרנט אחרי טעינה ראשונה.
- **מובייל תחילה**: מיועד לשימוש ב-iPhone Safari. כפתורים מינימום 44px, ניווט תחתון קבוע, viewport מוגדר נכון, `apple-mobile-web-app-capable`.
- **דו-לשוני מלא**: עברית (RTL) ואנגלית (LTR). כל טקסט עובר דרך מערכת i18n פנימית — אסור טקסט קשיח בקוד. החלפת שפה מהפכת את כל הפריסה כולל גרפים וניווט.
- **מצב כהה/בהיר**: כל הצבעים דרך CSS variables בלבד.
- **שמירה מקומית**: כל הנתונים ב-localStorage תחת מפתח אחד ראשי (`nutritionApp`). ייצוא/ייבוא JSON לגיבוי.
- **ללא ספריות חיצוניות בזמן ריצה**: גרפים ב-Canvas או SVG בקוד עצמי. אם משתמשים בפונט מ-Google Fonts — חובה fallback למקרה של חוסר אינטרנט.

## מבנה האפליקציה (5 טאבים)
1. **בית** — מצב פעיל (חיטוב/מסה) + תאריך תחילה + מונה ימים, סיכום משקל אחרון, שינוי שבועי, התקדמות ליעד, ארוחות היום.
2. **תפריט** — תפריט נפרד לכל מצב (חיטוב/מסה), עריכת ארוחות ופריטים, סימון ✓ לארוחה שנאכלה עם איפוס יומי, טבלת המרות גולמי↔מבושל + מחשבון המרה.
3. **מדידות** — הזנה שבועית: משקל, מותניים, בטן, יד, צוואר. גרפים לכל פרמטר + משולב, קווי יעד, שינוי שבועי עם צבע חכם לפי מצב (חיטוב: ירידה=ירוק; מסה: עלייה במשקל=ירוק), BMI ואחוז שומן לפי US Navy לגברים.
4. **אימונים** — יומן אימונים: תרגילים, סטים, חזרות, משקל לכל סט. היסטוריה לכל תרגיל, גרף משקל מקסימלי וגרף נפח (סטים×חזרות×משקל), מסך שיאים אישיים (PR) עם זיהוי אוטומטי ואנימציית חגיגה.
5. **הגדרות** — שפה, מצב כהה/בהיר, גובה (ל-BMI), יעדים, ייצוא/ייבוא JSON, איפוס נתונים עם אישור כפול.

## נוסחאות
- BMI = משקל(ק"ג) / גובה(מ')²
- אחוז שומן (US Navy, גברים): 495 / (1.0324 - 0.19077*log10(מותניים-צוואר) + 0.15456*log10(גובה)) - 450 (מידות בס"מ)
- PR = המשקל הגבוה ביותר שנרשם אי פעם לתרגיל

## סגנון עבודה
- עברית בתגובות אליי, קוד ותיעוד פנימי באנגלית
- אחרי כל שינוי — לוודא שהקובץ עדיין עצמאי ותקין (פתיחה ישירה בדפדפן)
- לא לשבור נתונים קיימים ב-localStorage: כל שינוי במבנה הנתונים חייב migration
- שינויים ממוקדים בלבד; לא לשכתב חלקים שעובדים

## תוכניות עתידיות
עטיפה ב-Capacitor לפרסום ב-App Store / Google Play — לכן לשמור על קוד נקי ומודולרי גם בתוך הקובץ היחיד.
